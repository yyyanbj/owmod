描述阶段规则

  • 每个玩家轮流描述自己的词语（不能直接说出）
  • 主持人可按 F 键跳过当前玩家
  • 时间到（30 秒）自动跳过
  • BOT 玩家自动跳过，不消耗时间
  • 所有玩家描述完成后进入投票阶段

5. VOTING 阶段 (投票阶段)

投票 HUD - 每个玩家独立显示
投票阶段
00:15
第 1 回合

按 [Q] 上一个，[E] 下一个

按 [F] 确认投票

你选择投票给：玩家 3 (Player3)

投票阶段规则

  • Q 键 (ULTIMATE)：选择上一个存活玩家
  • E 键 (ABILITY_2)：选择下一个存活玩家
  • F 键 (INTERACT)：确认投票给当前选择的玩家
  • 时间到（15 秒）或所有人投票后统计票数
  • BOT 自动投票给随机存活玩家
  • 平票时随机淘汰一个

7. GAME_OVER 阶段 (游戏结束)

游戏结束 HUD

游戏结束流程

  • 显示胜负结果大消息（平民胜利/卧底胜利）
  • 等待 10 秒让玩家查看结果
  • Debug 模式：清理所有 Bot
  • 重置所有游戏变量和玩家状态
  • 返回 WAITING 阶段，等待下一局

9. OverPy 代码结构

# 文件结构 src/ ├── main.opy # 主游戏入口，游戏循环和事件处理 ├── config.opy # 游戏常量定义（MIN_PLAYERS, UNDERCOVER_* ├── game_state.opy # 游戏状态机和状态转换函数 ├── word_assignment.opy # 词语分配逻辑 ├── description.opy # 描述阶段逻辑（nextSpeaker 等） ├── voting.opy # 投票系统和 Bot AI ├── hud.opy # HUD 界面函数（hudHeader, hudSubtext） └── wordbank.opy # 词库管理 # 全局变量声明 (main.opy) globalvar gameStatus # 当前游戏状态 globalvar targetState # 目标状态（用于状态转换） globalvar civilianWord # 平民词语 globalvar undercoverWord # 卧底词语 globalvar undercoverCount # 卧底数量 globalvar roundNumber # 当前回合 globalvar currentSpeaker # 当前描述玩家编号 globalvar timerValue # 倒计时值 globalvar debugMode # Debug 模式开关 globalvar debugBotCount # Debug 模式下 Bot 数量 playervar playerWord # 每个玩家的词语 playervar isUndercover # 是否卧底 playervar isAlive # 是否存活 playervar hasVoted # 是否已投票 playervar selectedOption # 当前选择的投票目标 playervar playerName # 玩家名字 playervar wordDisplayString # 词语显示字符串
